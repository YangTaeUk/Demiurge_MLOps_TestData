## 1. 어댑터 구현 현황 총괄

### 1.1 한눈에 보는 준비 상태

22종 어댑터는 **외부 접속자 관점에서 3단계 준비 등급**으로 나뉜다.

#### Tier 1 — 즉시 사용 가능 (6종)

`docker compose up -d` 또는 설치만으로 바로 파이프라인 연결 가능.

| 어댑터 | 카테고리 | 인프라 | 통합테스트 | push() | fetch() | 비고 |
|--------|---------|--------|-----------|--------|---------|------|
| **PostgreSQL** | RDBMS | Docker 제공 | 6/6 passed | ✅ | ✅ | E2E 검증 완료 |
| **MongoDB** | NoSQL | Docker 제공 | 5/5 passed | ✅ | ✅ | E2E 검증 완료 |
| **Kafka** | Streaming | Docker 제공 | 5/5 passed | ✅ | ❌ stub | E2E 검증 (push만) |
| **S3 (MinIO)** | Storage | Docker 제공 | 6/6 passed | ✅ | ✅ | E2E 검증 완료 |
| **SQLite** | RDBMS | 불필요 (로컬) | 없음 | ✅ | ✅ | 메모리/파일 DB |
| **Local FS** | Storage | 불필요 (로컬) | 없음 | ✅ | ✅ | 로컬 디스크 저장 |

#### Tier 2 — 코드 완성, 외부 인프라 필요 (11종)

모든 메서드가 구현되어 있으나, 해당 서비스를 직접 프로비저닝해야 한다. Docker Compose에 추가하거나 클라우드 서비스를 연결하면 즉시 동작한다.

| 어댑터 | 카테고리 | 필요 인프라 | push() | fetch() | 비동기 방식 | 비고 |
|--------|---------|-----------|--------|---------|-----------|------|
| **MySQL** | RDBMS | MySQL 서버 | ✅ | ✅ | 네이티브 async | 146줄 풀 구현 |
| **MSSQL** | RDBMS | SQL Server + ODBC 드라이버 | ✅ | ✅ | 네이티브 async | OS별 ODBC 설치 필요 |
| **Oracle** | RDBMS | Oracle DB | ✅ | ✅ | 네이티브 async | thin mode, BLOB 타입 |
| **BigQuery** | RDBMS | GCP 프로젝트 + 서비스 계정 | ✅ | ✅ | 동기 클라이언트 | 에뮬레이터 지원 |
| **Redis** | NoSQL | Redis 서버 | ✅ | ✅ | 네이티브 async | pipeline 벌크 지원 |
| **Elasticsearch** | NoSQL | ES 클러스터 | ✅ | ✅ | 네이티브 async | bulk API 지원 |
| **Cassandra** | NoSQL | Cassandra 클러스터 | ✅ | ✅ | **to_thread 래핑** | keyspace 자동 생성 |
| **HDFS** | Storage | Hadoop 클러스터 | ✅ | ✅ | **to_thread 래핑** | WebHDFS REST API |
| **FTP** | FileTransfer | FTP 서버 | ✅ | ✅ | 네이티브 async | upload/download 지원 |
| **SFTP** | FileTransfer | SSH 서버 | ✅ | ✅ | **to_thread 래핑** | 키 기반 인증 지원 |
| **MariaDB** | RDBMS | MariaDB 서버 | ✅ | ✅ | 네이티브 async | MySQL 상속 (27줄) |

#### Tier 3 — Producer 전용, fetch() 미구현 (5종)

Streaming 어댑터 5종은 **설계 의도에 따라 `push()` (발행)만 구현**되어 있다. `fetch()` (소비)는 의도적으로 stub이다. 이 프로젝트의 파이프라인 흐름이 `Generator → Handler → Adapter.push()` 단방향이기 때문이다.

| 어댑터 | 카테고리 | 필요 인프라 | push() | fetch() | subscribe() | 비동기 방식 |
|--------|---------|-----------|--------|---------|------------|-----------|
| **RabbitMQ** | Streaming | RabbitMQ 서버 | ✅ | ❌ stub | ⚠️ 부분 | 네이티브 async |
| **NATS** | Streaming | NATS 서버 | ✅ | ❌ stub | ⚠️ 부분 | 네이티브 async |
| **MQTT** | Streaming | MQTT 브로커 | ✅ | ❌ stub | ⚠️ 부분 | 네이티브 async |
| **Pulsar** | Streaming | Pulsar 클러스터 | ✅ | ❌ stub | ⚠️ 부분 | **to_thread 래핑** |
| **CockroachDB** | RDBMS | CockroachDB 서버 | ✅ | ✅ | — | 네이티브 async |

> CockroachDB는 코드 완성이지만 PostgreSQL 상속 래퍼(28줄)로 분류.

### 1.3 상속 관계와 실질 중복

2종의 어댑터는 부모 어댑터를 그대로 상속하며, DSN(연결 문자열) 프리픽스만 다르다.

| 어댑터 | 상속 대상 | 자체 코드 | 차이점 |
|--------|----------|----------|--------|
| **MariaDB** | MySQLAdapter | 27줄 | 포트 기본값 (3307), DSN 프리픽스 |
| **CockroachDB** | PostgreSQLAdapter | 28줄 | DSN 프리픽스 (`cockroachdb+asyncpg://`) |

부모 어댑터가 동작하면 이들도 그대로 동작한다. 별도 테스트는 불필요.

### 1.4 동기/비동기 드라이버 현황

대부분의 어댑터는 **네이티브 비동기 드라이버**를 사용하지만, 4종은 동기 드라이버를 `asyncio.to_thread()`로 감싸는 방식이다. 대량 I/O 시 스레드풀 병목이 발생할 수 있다.

| 구분 | 네이티브 비동기 | `asyncio.to_thread()` 래핑 |
|------|---------------|--------------------------|
| RDBMS | asyncpg, aiomysql, aiosqlite, aioodbc, oracledb | — |
| NoSQL | motor, redis, elasticsearch | **cassandra-driver** |
| Streaming | FastStream (kafka, rabbit, nats), aiomqtt | **pulsar-client** |
| Storage | aiofiles | **s3fs**, **fsspec (HDFS)** |
| FileTransfer | aioftp | **paramiko (SFTP)** |

> **성능 주의**: Cassandra, Pulsar, HDFS, SFTP — 동시 대량 요청 시 기본 스레드풀(8개) 포화 가능. `asyncio.get_event_loop().set_default_executor(ThreadPoolExecutor(max_workers=N))`으로 확장 가능.

### 1.5 Streaming 어댑터의 fetch() 미구현 — 설계 의도

```python
# 모든 Streaming 어댑터의 fetch() 구현 — 의도적 stub
async def fetch(self, query, limit=None) -> AsyncIterator[bytes]:
    return
    yield  # pragma: no cover — async generator 형식 유지용
```

**이유**: 이 프로젝트의 핵심 데이터 흐름은 **테스트 데이터 생성 → 대상 시스템에 주입**이다.

```
Generator ──→ HandlerChain ──→ Adapter.push()  ← 이 방향만 구현
                                    │
                            Kafka / RabbitMQ / NATS / MQTT / Pulsar
                                    │
                            외부 Consumer가 소비  ← 이 프로젝트의 범위 밖
```

Streaming 시스템에서 데이터를 **소비하는 것은 외부 애플리케이션의 역할**이다. 이 프로젝트는 소비 측이 아닌 주입 측(Producer)에 집중한다.

> **Consumer가 필요한 경우**: 각 브로커의 네이티브 Consumer 클라이언트를 직접 사용하거나, FastStream의 `@broker.subscriber()` 데코레이터를 활용.

## 4. RDBMS 어댑터

### 4.1 PostgreSQL

> **Tier 1** — Docker 제공, 통합테스트 완료, E2E 검증 완료

**레지스트리 키**: `postgresql` | **드라이버**: asyncpg (네이티브 async) | **포트**: 5434 (Docker)

```python
import asyncio
from demiurge_testdata.adapters.rdbms.postgresql import PostgreSQLAdapter
from demiurge_testdata.schemas.config import RDBMSAdapterConfig

async def main():
    config = RDBMSAdapterConfig(
        host="localhost",
        port=5434,
        user="testdata",
        password="testdata_dev",
        database="testdata",
        # pool_size=5,       # 연결 풀 크기 (기본 5)
        # pool_overflow=10,  # 최대 초과 연결 (기본 10)
    )
    adapter = PostgreSQLAdapter(config=config)
    await adapter.connect()

    # 헬스체크
    assert await adapter.health_check() is True

    # 테이블 생성
    await adapter.create_table("users", {
        "name": "VARCHAR(100)",
        "age": "INTEGER",
        "email": "VARCHAR(200)",
    })

    # 대량 삽입 — 주의: asyncpg는 Python 타입을 엄격하게 매핑
    records = [
        {"name": "Alice", "age": 30, "email": "alice@example.com"},  # age: int (str 불가)
        {"name": "Bob", "age": 25, "email": "bob@example.com"},
    ]
    count = await adapter.bulk_insert("users", records)

    # SQL 실행
    result = await adapter.execute_sql("SELECT * FROM users WHERE age > $1", [20])

    # 바이트 데이터 push/fetch (파이프라인용)
    await adapter.push(b'{"data": "test"}', {
        "table": "pipeline_data",
        "format": "json",
        "record_count": 1,
    })

    await adapter.disconnect()

asyncio.run(main())
```

## 6. Streaming 어댑터

> **중요**: 5종 Streaming 어댑터는 모두 **Producer 전용**이다. `push()` (메시지 발행)는 완전 구현되어 있으나, `fetch()` (메시지 소비)는 의도적으로 구현하지 않았다 (stub). 이 프로젝트의 파이프라인은 테스트 데이터를 메시지 브로커에 **주입**하는 것이 목적이며, 소비는 외부 시스템의 역할이다.

### 6.1 Kafka

> **Tier 1** — Docker 제공, 통합테스트 완료, E2E 검증 완료 | **push() 전용**

**레지스트리 키**: `kafka` | **드라이버**: FastStream[kafka] (네이티브 async) | **포트**: 9092

```python
import asyncio
from demiurge_testdata.adapters.streaming.kafka import KafkaAdapter
from demiurge_testdata.schemas.config import StreamAdapterConfig

async def main():
    config = StreamAdapterConfig(
        host="localhost",
        port=9092,
        topic="my-events",
    )
    adapter = KafkaAdapter(config=config)
    await adapter.connect()

    # 메시지 발행 (push)
    await adapter.push(b'{"event": "order_created"}', {
        "topic": "my-events",
        "format": "json",
        "record_count": 1,
    })

    # fetch()는 stub — 소비가 필요하면 외부 Consumer 사용
    # async for msg in adapter.fetch({"topic": "my-events"}):  ← 동작하지 않음

    await adapter.disconnect()

asyncio.run(main())
```

**설정 파라미터 (`StreamAdapterConfig`)** — Kafka, RabbitMQ, MQTT, Pulsar 공통:

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|-------|------|
| `host` | str | `"localhost"` | 브로커 호스트 |
| `port` | int | `9092` | 브로커 포트 |
| `topic` | str | `"testdata"` | 기본 토픽/큐 |

### 9.4 Python API 기반 실행

```python
import asyncio
from demiurge_testdata.core.config import GeneratorConfig
from demiurge_testdata.core.pipeline import DataPipeline
from demiurge_testdata.core.registry import format_registry, generator_registry
from demiurge_testdata.handlers.chain import HandlerChain

# 모듈 등록 (import 시 레지스트리에 자동 등록됨)
import demiurge_testdata.generators.relational.home_credit  # noqa: F401
import demiurge_testdata.handlers.formats.json_handler  # noqa: F401
from demiurge_testdata.adapters.rdbms.postgresql import PostgreSQLAdapter
from demiurge_testdata.schemas.config import RDBMSAdapterConfig


async def main():
    # 1) Generator — 합성 레코드로 생성
    config = GeneratorConfig(type="home_credit", shuffle=True)
    gen_cls = generator_registry.get_class("home_credit")
    generator = gen_cls(config, records=sample_records)

    # 2) HandlerChain
    format_handler = format_registry.create("json")
    handler_chain = HandlerChain(format_handler=format_handler)

    # 3) Adapter
    adapter = PostgreSQLAdapter(config=RDBMSAdapterConfig(
        host="localhost", port=5434,
        user="testdata", password="testdata_dev", database="testdata",
    ))
    await adapter.connect()

    # 4) Pipeline 실행
    pipeline = DataPipeline(
        generator=generator,
        handler_chain=handler_chain,
        adapter=adapter,
        batch_size=1000,
    )
    metrics = await pipeline.run_batch()

    print(f"레코드: {metrics.total_records}")
    print(f"바이트: {metrics.total_bytes}")
    print(f"처리량: {metrics.records_per_second:.0f} records/sec")

    await adapter.disconnect()

asyncio.run(main())
```

## 11. 문제 해결

### 11.1 연결 실패

| 증상 | 원인 | 해결 |
|------|------|------|
| `Connection refused` | Docker 서비스 미시작 | `docker compose up -d` |
| `Authentication failed` | 자격증명 불일치 | `.env` 및 config 확인 |
| `Timeout` | 서비스 초기화 미완료 | `docker compose ps`로 `healthy` 확인 후 재시도 |
| `No module named 'asyncpg'` | 선택적 의존성 미설치 | `pip install "demiurge-testdata[rdbms]"` |
| `ODBC Driver not found` | MSSQL ODBC 드라이버 미설치 | OS별 ODBC Driver 18 설치 |
| `Broker is not connected` | Kafka 브로커 미응답 | Zookeeper → Kafka 순서로 시작 확인 |

### 11.2 포트 충돌

```bash
# 사용 중인 포트 확인
ss -tlnp | grep -E '5434|27017|9092|9000'

# 포트 변경 (.env 파일)

```

### 11.3 데이터 초기화

```bash
# 전체 초기화 (볼륨 삭제)
docker compose down -v

# PostgreSQL만 초기화
docker compose exec postgres psql -U testdata -d testdata \
    -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public;"

# MongoDB만 초기화
docker compose exec mongodb mongosh -u testdata -p testdata_dev \
    --authenticationDatabase admin testdata --eval "db.dropDatabase()"

# MinIO 버킷 비우기
mc rm --recursive --force local/testdata/
```

### 11.4 asyncpg 타입 에러

asyncpg는 Python 타입을 PostgreSQL 타입에 엄격하게 매핑한다:

```python
# 잘못된 예 — str을 INTEGER 컬럼에 삽입
records = [{"name": "Alice", "age": "30"}]  # age가 str → TypeError
await adapter.bulk_insert("users", records)

# 올바른 예 — Python int 사용
records = [{"name": "Alice", "age": 30}]    # age가 int → OK
await adapter.bulk_insert("users", records)
```

### 11.5 동기 드라이버 성능 이슈

Cassandra, Pulsar, HDFS, SFTP는 `asyncio.to_thread()` 래핑으로 동작한다. 동시 요청이 많을 때:

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

# 기본 스레드풀(8개)을 확장
loop = asyncio.get_event_loop()
loop.set_default_executor(ThreadPoolExecutor(max_workers=32))
```

# Zookeeper가 먼저 healthy 상태인지 확인
