## 1. 데이터 수집·시딩 아키텍처 개요

### 1.1 왜 시딩이 필요한가

기존 `push(bytes)` 파이프라인은 **Handler가 직렬화한 바이트**를 어댑터에 전달한다. 어댑터는 이 바이트를 `raw_data` 테이블이나 단일 문서에 저장할 뿐이다:

```
현재 push(): INSERT INTO raw_data (data) VALUES (b'[{"SK_ID":100000,...}]')
  → 외부 시스템이 raw_data 테이블에서 opaque bytes 조회 → 쓸모없음
```

시딩은 이와 다르다. **데이터셋 스키마에 맞는 구조화된 형태**로 인프라에 적재한다:

```
시딩: INSERT INTO home_credit (SK_ID_CURR, AMT_INCOME_TOTAL, ...) VALUES (100000, 50000.0, ...)
  → 외부 시스템이 SELECT * FROM home_credit WHERE TARGET=1 → 정상 쿼리 가능
```

### 1.2 전체 아키텍처

```
┌─────────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ Kaggle API  │───▶│ data/raw/│───▶│  Validation  │───▶│ SeedPipeline │───▶│ 구조화 적재   │
│ (kaggle CLI)│    │ CSV 파일  │    │ (스키마 검증) │    │ (core/seed)  │    │ (인프라별)    │
└─────────────┘    └──────────┘    └──────────────┘    └──────────────┘    └──────────────┘
       ↑                                                      │
  configs/datasets/                                           ├─ RDBMS: create_table → bulk_insert
  manifest.yaml                                               ├─ NoSQL: insert_documents
                                                              ├─ Storage: HandlerChain.encode → write
                                                              └─ Streaming: HandlerChain.encode → publish
```

### 1.3 push(bytes) vs 시딩 비교

| 구분 | push(bytes) | 구조화 시딩 |
|------|-------------|-----------|
| **데이터 형태** | 직렬화된 바이트 덩어리 | 컬럼별 분리된 레코드 |
| **저장 위치** | `raw_data` 테이블 / 단일 문서 | 데이터셋 이름의 전용 테이블/컬렉션 |
| **외부 쿼리** | 불가능 (opaque bytes) | SQL, MongoDB 쿼리, S3 SELECT 가능 |
| **용도** | 포맷·압축 호환성 테스트 | 외부 시스템 데이터 소비 |
| **Handler 개입** | 필수 (직렬화+압축) | RDBMS/NoSQL: 불필요, Storage/Streaming: 필요 |

### 2.2 KaggleDownloader 모듈

새 모듈: `src/demiurge_testdata/data/downloader.py`

```python
"""Kaggle 데이터셋 다운로드 및 검증"""

from __future__ import annotations

import subprocess
from pathlib import Path

from demiurge_testdata.core.exceptions import DataDownloadError


class KaggleDownloader:
    """Kaggle CLI를 통한 데이터셋 다운로드 관리자"""

    def __init__(self, data_dir: Path = Path("data/raw")):
        self._data_dir = data_dir

    def download(self, kaggle_id: str, dataset_name: str) -> Path:
        """
        Kaggle 데이터셋 다운로드.

        Args:
            kaggle_id: Kaggle 데이터셋 식별자 (예: "home-data-for-ml-course/home-credit-default-risk")
            dataset_name: 로컬 디렉토리명 (예: "home_credit")

        Returns:
            다운로드된 파일 경로 (data/raw/{dataset_name}/)
        """
        dest = self._data_dir / dataset_name
        dest.mkdir(parents=True, exist_ok=True)

        result = subprocess.run(
            [
                "kaggle", "datasets", "download",
                "-d", kaggle_id,
                "-p", str(dest),
                "--unzip",
            ],
            capture_output=True,
            text=True,
        )
        if result.returncode != 0:
            raise DataDownloadError(
                f"Kaggle download failed for {kaggle_id}: {result.stderr}"
            )
        return dest

    def download_competition(self, competition: str, dataset_name: str) -> Path:
        """Kaggle Competition 데이터셋 다운로드 (별도 API)"""
        dest = self._data_dir / dataset_name
        dest.mkdir(parents=True, exist_ok=True)

        result = subprocess.run(
            [
                "kaggle", "competitions", "download",
                "-c", competition,
                "-p", str(dest),
                "--force",
            ],
            capture_output=True,
            text=True,
        )
        if result.returncode != 0:
            raise DataDownloadError(
                f"Kaggle competition download failed for {competition}: {result.stderr}"
            )
        return dest

    def verify(self, dataset_name: str, expected_files: list[str]) -> bool:
        """다운로드된 파일 존재 여부 검증"""
        dest = self._data_dir / dataset_name
        for filename in expected_files:
            if not (dest / filename).exists():
                return False
        return True
```

### 3.2 매니페스트 필드 설명

| 필드 | 설명 |
|------|------|
| `kaggle_id` | Kaggle API 식별자 (`owner/dataset` 또는 `competition-slug`) |
| `kaggle_type` | `dataset` (일반) 또는 `competition` (대회) — 다운로드 API가 다름 |
| `category` | 6가지 카테고리: relational, document, event, iot, text, geospatial |
| `files` | 다운로드 후 존재해야 하는 파일 목록 |
| `primary_file` | 시딩에 사용할 메인 CSV 파일 |
| `seed_target` | 시딩 대상 어댑터 (postgresql, mongodb, kafka, s3, ...) |
| `seed_table` / `seed_collection` / `seed_topic` / `seed_key` | 어댑터별 대상 리소스명 |
| `size_mb` | 예상 디스크 용량 (MB) |
| `record_count` | 예상 레코드 수 |

## 4. 다운로드 워크플로

### 4.1 전체 흐름

```
1. manifest.yaml 로드
2. 데이터셋 선택 (--dataset 또는 전체)
3. Kaggle API 호출 (kaggle datasets download / competitions download)
4. data/raw/{dataset}/ 에 압축 해제
5. 파일 존재 검증 (manifest의 files 목록 대조)
6. 기본 무결성 검사 (CSV 헤더 파싱, 행 수 확인)
```

### 4.2 선택적 다운로드

```bash
# 단일 데이터셋
python -m demiurge_testdata download --dataset home_credit

# 카테고리별
python -m demiurge_testdata download --category relational
```

## 5. 시딩 파이프라인 설계

### 5.1 핵심 클래스: SeedPipeline

새 모듈: `src/demiurge_testdata/core/seed.py`

```python
"""구조화된 데이터 시딩 파이프라인"""

from __future__ import annotations

from pathlib import Path
from typing import Any

from demiurge_testdata.adapters.base import (
    BaseNoSQLAdapter,
    BaseRDBMSAdapter,
    BaseStorageAdapter,
    BaseStreamAdapter,
)
from demiurge_testdata.core.registry import adapter_registry
from demiurge_testdata.handlers.chain import HandlerChain


class SeedPipeline:
    """
    CSV 원본 데이터를 인프라에 구조화된 형태로 시딩한다.

    push(bytes)와 달리, 어댑터 카테고리에 따라 적재 전략이 달라진다:
    - RDBMS: create_table(schema) → bulk_insert(records)
    - NoSQL: insert_documents(collection, documents)
    - Storage: HandlerChain.encode() → adapter.write(key, bytes)
    - Streaming: HandlerChain.encode() → adapter.publish(topic, msg)
    """

    def __init__(
        self,
        adapter_type: str,
        adapter_config: dict[str, Any],
        handler_chain: HandlerChain | None = None,
    ):
        self._adapter = adapter_registry.create(adapter_type, config=adapter_config)
        self._handler_chain = handler_chain

    async def seed_rdbms(
        self,
        table: str,
        columns: list[dict[str, str]],
        records: list[dict],
        batch_size: int = 5000,
    ) -> int:
        """
        RDBMS에 구조화된 테이블을 생성하고 레코드를 벌크 삽입한다.

        1. create_table(table, columns)
        2. for batch in chunks(records, batch_size):
               bulk_insert(table, batch)

        기존 코드 재활용: BaseRDBMSAdapter.create_table(), bulk_insert()
        """
        if not isinstance(self._adapter, BaseRDBMSAdapter):
            raise TypeError(f"RDBMS 시딩에는 BaseRDBMSAdapter가 필요합니다 (got {type(self._adapter).__name__})")
        adapter: BaseRDBMSAdapter = self._adapter
        async with adapter:
            await adapter.create_table(table, columns)

            total = 0
            for i in range(0, len(records), batch_size):
                batch = records[i : i + batch_size]
                await adapter.bulk_insert(table, batch)
                total += len(batch)
            return total

    async def seed_nosql(
        self,
        collection: str,
        documents: list[dict],
        batch_size: int = 1000,
    ) -> int:
        """
        NoSQL에 JSON 문서를 삽입한다.

        기존 코드 재활용: BaseNoSQLAdapter.insert_documents()
        """
        if not isinstance(self._adapter, BaseNoSQLAdapter):
            raise TypeError(f"NoSQL 시딩에는 BaseNoSQLAdapter가 필요합니다 (got {type(self._adapter).__name__})")
        adapter: BaseNoSQLAdapter = self._adapter
        async with adapter:
            total = 0
            for i in range(0, len(documents), batch_size):
                batch = documents[i : i + batch_size]
                count = await adapter.insert_documents(collection, batch)
                total += count
            return total

    async def seed_storage(
        self,
        key: str,
        records: list[dict],
    ) -> int:
        """
        Storage에 직렬화된 파일을 업로드한다.

        HandlerChain.encode()로 Parquet/JSON 등으로 직렬화 후 write().
        """
        if not isinstance(self._adapter, BaseStorageAdapter):
            raise TypeError(f"Storage 시딩에는 BaseStorageAdapter가 필요합니다 (got {type(self._adapter).__name__})")
        adapter: BaseStorageAdapter = self._adapter
        if self._handler_chain is None:
            raise ValueError("Storage 시딩에는 HandlerChain이 필요합니다")

        async with adapter:
            encoded = await self._handler_chain.encode(records)
            await adapter.write(key, encoded)
            return len(records)

    async def seed_streaming(
        self,
        topic: str,
        records: list[dict],
        batch_size: int = 100,
    ) -> int:
        """
        Streaming 브로커에 메시지를 발행한다.

        HandlerChain.encode()로 직렬화 후 배치 발행.
        """
        if not isinstance(self._adapter, BaseStreamAdapter):
            raise TypeError(f"Streaming 시딩에는 BaseStreamAdapter가 필요합니다 (got {type(self._adapter).__name__})")
        adapter: BaseStreamAdapter = self._adapter
        if self._handler_chain is None:
            raise ValueError("Streaming 시딩에는 HandlerChain이 필요합니다")

        async with adapter:
            total = 0
            for i in range(0, len(records), batch_size):
                batch = records[i : i + batch_size]
                encoded = await self._handler_chain.encode(batch)
                await adapter.publish(topic, encoded)
                total += len(batch)
            return total
```

### 5.2 카테고리별 시딩 메서드 비교

| 카테고리 | 시딩 메서드 | Handler 필요 | 결과물 |
|---------|-----------|-------------|--------|
| **RDBMS** | `create_table()` → `bulk_insert()` | 불필요 | 컬럼 기반 테이블 |
| **NoSQL** | `insert_documents()` | 불필요 | JSON 문서 컬렉션 |
| **Storage** | `HandlerChain.encode()` → `write()` | 필수 (Parquet 등) | 구조화 파일 (S3 객체) |
| **Streaming** | `HandlerChain.encode()` → `publish()` | 필수 (JSON 등) | 토픽 메시지 |

| RDBMS | 테이블 (예: `home_credit`) | PostgreSQL:5434 직접 SQL 쿼리 | REST `GET /data/postgresql/query` |

| Storage | S3 객체 (예: `hm/*.parquet`) | MinIO:9000 직접 다운로드 | REST `GET /data/s3/query` |
| Streaming | 토픽 메시지 (예: `ieee-fraud-events`) | Kafka:9092 Consumer Group | gRPC `StreamData` RPC |
